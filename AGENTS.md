# Инструкции для Codex

Перед любым изменением навигации, экрана, маршрута или сущности прочитай:

1. `docs/ARCHITECTURE.md`
2. `docs/SCREENS.yaml`
3. `docs/ACTIONS.yaml`
4. `docs/ROUTES.yaml`
5. `docs/DECISIONS.md`
6. подробную спецификацию экрана в `docs/screens/`

## Обязательные правила

- Не создавать дублирующие экраны.
- У каждого экрана должен быть уникальный `screen_id`.
- У каждой кнопки и кликабельной строки должен быть `action_id`.
- У каждого `action_id` должен быть существующий целевой экран или системное действие.
- У каждого детального экрана должен быть `back_fallback` для прямого deep link.
- Нижняя навигация содержит ровно пять разделов: Главная, Играть, Чаты, Клубы, Профиль.
- «Играть» — публичный каталог. «Профиль → Мои игры» — личный центр участия и управления.
- Одноразовая игра, тренировка, турнир, сезон и тур — разные сущности.
- «Ротация пяти игроков» — формат игры. Количество кортов задаётся на уровне конкретной игры или игрового дня сезона.
- После изменения интерфейса синхронно обновлять `SCREENS.yaml`, `ACTIONS.yaml`, `ROUTES.yaml` и соответствующую спецификацию экрана.
- Новое спорное решение сначала записывать в `DECISIONS.md`.

## Запреты

- Не добавлять шестую нижнюю вкладку.
- Не размещать туры внутри обычных игр.
- Не создавать отдельные копии страницы игры для Главной, Календаря, Чатов и Профиля.
- Не прятать критичное действие только в меню `...`.
- Не оставлять кнопку без описанного перехода или результата.
