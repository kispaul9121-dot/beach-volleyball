# VolleyPlay

Архитектура мобильного приложения для пляжного волейбола.

## С чего начинать

1. Откройте [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md) — это главная точка входа.
2. Посмотрите [`docs/APP_MAP.md`](docs/APP_MAP.md) — карта разделов и переходов.
3. Для конкретного экрана найдите его в [`docs/SCREENS.yaml`](docs/SCREENS.yaml), затем откройте подробную спецификацию в `docs/screens/`.
4. Каждая кнопка описана в [`docs/ACTIONS.yaml`](docs/ACTIONS.yaml).
5. Реальные URL и возвраты назад описаны в [`docs/ROUTES.yaml`](docs/ROUTES.yaml).
6. Принятые решения и причины хранятся в [`docs/DECISIONS.md`](docs/DECISIONS.md).

## Главный принцип

У каждой сущности есть один канонический экран. Главная, календарь, профиль, чат и уведомления могут вести на него, но не создают копии экрана.

## Пять основных разделов

- Главная
- Играть
- Чаты
- Клубы
- Профиль

## Текущий статус

Архитектура находится на стадии проектирования. Первыми подробно описаны:

- Профиль → Мой календарь
- Профиль → Мои игры
- публичный каталог «Играть»
- детали одноразовой игры
- создание и управление одноразовой игрой
- турниры, сезоны и формат «ротация пяти игроков»
