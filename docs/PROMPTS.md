# Промпты для Codex

## Реализация нового экрана

Прочитай `AGENTS.md`, `docs/ARCHITECTURE.md`, `docs/SCREENS.yaml`, `docs/ACTIONS.yaml`, `docs/ROUTES.yaml`, `docs/UI_RULES.md` и спецификацию нужного экрана.

Реализуй экран `<screen_id>` без создания новых маршрутов и без дублирования существующих компонентов. Все кликабельные элементы должны соответствовать `ACTIONS.yaml`. Реализуй loading, empty, error и offline состояния. Проверь back fallback и deep links.

## Проверка навигации

Проверь `SCREENS.yaml`, `ACTIONS.yaml` и `ROUTES.yaml`:

- каждый source и destination существует;
- у каждого вложенного экрана есть back fallback;
- нет двух screen_id с одинаковой целью;
- нет маршрутов без экрана;
- нет кнопок без действия;
- роли имеют правильные разрешения.

Составь список ошибок и исправь документацию до изменения кода.

## Обновление архитектуры

Перед изменением зафиксируй решение в `DECISIONS.md`. Затем синхронно обнови реестр экранов, действий, маршрутов, Mermaid-карту и подробную спецификацию экрана.
