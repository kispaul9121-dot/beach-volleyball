# Кэмпы

- Screen ID: `camps.main`
- Route: `/camps`
- Bottom tab: `Кэмпы`
- Management contract: `docs/MANAGEMENT_CENTER.yaml`

## Три состояния одного раздела

```text
/camps                 → подбор
/camps?mode=manage     → управление
/camps?mode=archive    → архив управления
```

По умолчанию открыта публичная витрина кэмпов. Кнопка `Режим управления` переключает текущую выдачу на кэмпы, которыми управляет active actor. При включённом режиме label меняется на `Вернуться к подбору`.

## Режим подбора

- поиск по направлению, дате, уровню и цене;
- сортировка от ближайшей даты к более поздней;
- карточка показывает даты, направление, автора, тип, заполненность, цену или ориентировочный бюджет;
- публичная страница использует единое действие `Вступить`;
- жест открытия архива здесь не работает.

## Режим управления

Показывается один список текущих собственных или доступных по роли кэмпов. Отдельных кнопок или вкладок `Активные` и `Завершённые` нет.

В списке могут находиться черновики, опубликованные кэмпы, записи с заявками, оплатами или документами, требующими обработки, а также уже идущие кэмпы. Состояния доступны как фильтры списка:

```text
Все · Черновики · Требуют действий · Опубликованы · Идут
```

Нажатие открывает `/tours/:tourId/manage`.

Основная кнопка:

```text
+ Создать кэмп
```

Кэмп может создать player, trainer или organization actor. После создания пользователь возвращается в режим управления кэмпами.

## Архив через протягивание вниз

Завершённые кэмпы считаются архивными и не показываются в текущем списке. Архив скрыт над верхней границей списка; постоянной кнопки, вкладки или пункта нижней навигации нет.

Жест работает только при `mode=manage`, когда список находится в самом верху:

1. после протяжки на 24 dp появляется `Архив` и подсказка `Потяните ещё, чтобы открыть`;
2. после 72 dp текст меняется на `Отпустите, чтобы открыть`, один раз срабатывает haptic;
3. отпускание раньше порога возвращает список;
4. отпускание после порога открывает `/camps?mode=archive`.

Pull-to-refresh в режиме управления отключён. Обновление выполняется при фокусе, восстановлении сети, realtime-событиях и отдельном повторе после ошибки.

Для VoiceOver/TalkBack у списка есть custom action `Открыть архив`. При Reduce Motion эластичное растяжение заменяется статичным индикатором и сменой текста без изменения порогов.

После появления первого завершённого кэмпа один раз допускается подсказка:

```text
Потяните список вниз, чтобы открыть архив
```

## Режим архива

Архив остаётся состоянием той же нижней вкладки:

```text
/camps?mode=archive
```

В шапке показываются `←` и `Архив кэмпов`. Список содержит только завершённые управляемые кэмпы и сортируется от недавно завершённых к более старым. Кнопка создания и фильтры текущих статусов скрыты.

Карточка открывает существующую историю управления или read-only представление по текущим правам. Назад возвращает в `/camps?mode=manage` с восстановлением actor, фильтров и позиции списка управления.

Архивирование не удаляет кэмп, участников, платежи и связанные данные. Ручное архивирование пока не утверждено.

## Нажатие на карточку в подборе

Используется `dynamic.catalog_entity_entry`:

- авторизованный менеджер → `/tours/:tourId/manage`;
- другой пользователь → `/tours/:tourId`;
- право повторно проверяется сервером.

## Личные участия

Личные записи находятся в `Вся моя активность` с фильтром `Кэмпы` и вкладками `Предстоящие · Прошедшие`.

## Пустые состояния

Управление:

```text
Вы пока не создавали кэмпы
[Создать кэмп]
```

Архив:

```text
В архиве пока нет завершённых кэмпов
```

## Возврат

После detail/manage/create восстанавливаются режим, actor-фильтр, фильтры и позиция списка. После архива восстанавливается предыдущее состояние режима управления.
