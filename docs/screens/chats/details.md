# Детали чата

- Screen ID: `chat.details`
- Route: `/chats/:chatId`
- Entity sections: `docs/ENTITY_SECTIONS.yaml`

## Шапка

- название и тип контекста;
- actor-профиль пользователя;
- игроки и организаторы чата;
- уведомления / mute;
- `Открыть событие / клуб / тур`;
- меню согласно разрешениям.

Отдельного контекста `Группа` в MVP нет.

## Сообщения

- текст;
- ответы;
- вложения;
- системные события;
- объявления;
- закреплённые сообщения;
- статусы отправки и повтор.

## Контекстные системные события

Примеры:

- изменено время;
- изменено место;
- изменена видимость события;
- опубликован раунд;
- назначен корт;
- требуется оплата;
- результат исправлен;
- игрок добавлен или удалён;
- событие завершено или отменено.

Системное событие содержит action_id и ведёт на нужный экран.

## Права

- подтверждённый игрок может писать согласно политике чата;
- приглашённый или заявитель без подтверждения не получает доступ;
- организатор может отправлять объявления и модерировать;
- сотрудник действует согласно entity role;
- read-only чат ясно показывает причину;
- потеря participation или membership пересчитывает доступ сервером.

## Связь с сущностью

Чат не копирует полную карточку. Нажатие открывает канонический экран события. Внутри события пользователь переходит к разделам `Игроки`, расписанию, таблице, результатам и статистике согласно типу и формату.

## Завершённое событие

По политике события чат:

- остаётся доступным;
- становится read-only;
- архивируется после срока хранения.

История не превращается в постоянную социальную группу.

## Состояния

- loading history;
- empty;
- read-only;
- removed from chat;
- context cancelled / archived;
- message failed;
- offline queue;
- permission denied.
