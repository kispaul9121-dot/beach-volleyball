# Детали чата

- Screen ID: `chat.details`
- Route: `/chats/:chatId`

## Шапка

- название и тип контекста;
- actor-профиль пользователя;
- участники;
- уведомления / mute;
- `Открыть событие / группу / клуб / тур`;
- меню согласно разрешениям.

## Сообщения

- текст;
- ответы;
- вложения;
- системные события;
- объявления;
- закреплённые сообщения;
- статусы отправки и повтор.

## Контекстные системные события

Примеры:

- изменено время;
- опубликован раунд;
- назначен корт;
- требуется оплата;
- результат исправлен;
- участник добавлен или удалён.

Системное событие содержит action_id и ведёт на нужный экран.

## Права

- участник может писать согласно политике чата;
- организатор может отправлять объявления и модерировать;
- сотрудник действует согласно entity role;
- read-only чат ясно показывает причину;
- потеря participation / membership пересчитывает доступ сервером.

## Связь с сущностью

Чат не копирует полную карточку. Нажатие открывает канонический экран. Если пользователь организатор — на публичном экране доступна кнопка управления.

## Состояния

- loading history;
- empty;
- read-only;
- removed from chat;
- context cancelled / archived;
- message failed;
- offline queue;
- permission denied.
