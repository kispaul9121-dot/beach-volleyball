# Кабинет организации — Активности

- Parent Screen ID: `club.manage`
- Route state: `/clubs/:clubId/manage?tab=activities`
- Source of truth: `docs/ORGANIZATION_ADMIN.yaml`

## Назначение

Единый рабочий список всех активностей организации без создания пяти параллельных административных разделов.

## Категории

- Все;
- Игры;
- Тренировки;
- Турниры;
- Сезоны;
- Туры.

## Статусы

- Требуют действий;
- Предстоящие;
- Идут сейчас;
- Черновики;
- Завершённые;
- Отменённые;
- Архив.

По умолчанию открывается `Требуют действий`, если такие элементы существуют; иначе — `Предстоящие`.

## Карточка активности

Показывает:

- тип и формат;
- название;
- дату или период;
- площадку;
- профиль-создатель;
- ответственного сотрудника;
- состояние регистрации;
- участников X из Y;
- задолженности;
- незакрытые результаты;
- ближайшую проблему;
- одно главное действие.

Главное действие:

- черновик → `Продолжить`;
- опубликовано без проблем → `Управлять`;
- есть заявки → `Проверить заявки`;
- есть неоплаты → `Открыть задолженности`;
- не сформирован раунд → `Сформировать раунд`;
- отсутствуют результаты → `Внести результаты`.

Вся карточка открывает канонический кабинет сущности. Быстрое действие ведёт непосредственно в нужный раздел кабинета.

## Создание

Кнопка `Создать` открывает только сущности, которые доступны организации и текущему сотруднику. Каждый мастер получает:

- `actorId` организации;
- `returnTo=/clubs/:clubId/manage?tab=activities`;
- выбранную площадку, если установлен фильтр;
- автора-сотрудника для аудита.

## Фильтры

- категория;
- статус;
- дата;
- площадка;
- ответственный;
- наличие заявок;
- наличие задолженностей;
- наличие конфликтов;
- созданные мной;
- назначенные мне.

## Массовые действия

Разрешены только безопасные операции:

- назначить ответственного;
- отправить объявление выбранным событиям;
- экспортировать список;
- архивировать уже завершённые элементы.

Массовая отмена, изменение цены, перенос и публикация раундов запрещены без отдельного подтверждаемого процесса для каждой сущности.

## Поиск

По названию, ID, площадке, сотруднику или участнику. Поиск участника доступен только ролям с `organization_people_view`.

## Различия по роли

- владелец и администратор видят все сущности;
- менеджер событий видит созданные им и назначенные ему сущности;
- тренер видит назначенные тренировки, группы и лагеря;
- финансы видят финансовое состояние без доступа к редактированию спортивного содержания;
- viewer получает read-only список.

## Состояния

- нет активностей — предложить создание;
- нет результатов по фильтру — сбросить фильтры;
- черновик принадлежит другому сотруднику;
- сотрудник потерял назначение;
- часть сущностей скрыта из-за прав;
- offline read-only;
- синхронизация после параллельного изменения.
