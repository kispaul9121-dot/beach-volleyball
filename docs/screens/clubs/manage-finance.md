# Кабинет организации — Финансы

- Parent Screen ID: `club.manage`
- Route state: `/clubs/:clubId/manage?tab=finance`
- Source of truth: `docs/ORGANIZATION_ADMIN.yaml`
- Permission: `finance_view`

## Назначение

Операционный финансовый кабинет организации: поступления, задолженности, возвраты, выплаты и отчёты. Он не заменяет бухгалтерскую систему и не позволяет вручную переписывать историю транзакций.

## Верхняя сводка

- поступления за период;
- ожидаемые платежи;
- просроченные задолженности;
- возвраты в обработке;
- доступно к выплате;
- следующая выплата;
- неуспешные транзакции.

Суммы всегда показывают валюту и период. Пользователь может открыть методику расчёта.

## Разделы

- Обзор;
- Транзакции;
- Задолженности;
- Возвраты;
- Выплаты;
- Отчёты;
- Настройки выплат — только соответствующей роли.

## Транзакции

Фильтры:

- период;
- статус;
- тип активности;
- конкретная сущность;
- площадка;
- способ оплаты;
- сотрудник-инициатор;
- валюта.

Карточка транзакции показывает order, плательщика, источник, сумму, комиссию, net, статус и время. Переход открывает канонический `payment.details`.

## Задолженности

Показываются:

- участник;
- сущность;
- сумма;
- срок;
- попытки оплаты;
- статус участия;
- разрешённые действия.

Действия: отправить напоминание, открыть участие, отметить внешнюю оплату при поддерживаемой политике, отменить заказ, инициировать возврат или исключение. Отметка внешней оплаты требует подтверждения и аудита.

## Возвраты

Состояния:

- запрос создан;
- требует решения;
- одобрен;
- отправлен провайдеру;
- выполнен;
- отклонён;
- частичный;
- спор.

Организатор видит применимую политику, фактическую сумму и влияние на участие. Возврат создаёт новую финансовую операцию.

## Выплаты

- статус payout account;
- доступный баланс;
- ожидающие суммы;
- график выплат;
- история;
- ошибки проверки или банковского перевода.

Изменение реквизитов требует повторной аутентификации и может временно приостановить выплаты.

## Отчёты

Предустановки:

- по периоду;
- по активности;
- по площадке;
- по тренеру;
- по налоговой категории, когда поддерживается;
- задолженности;
- возвраты;
- выплаты.

Экспорт через `organization_admin.export_financial_report` фиксирует пользователя, actor, фильтры и время.

## Права

- `finance_view` — просмотр разрешённых финансовых данных;
- `finance_manage` — возвраты, внешние оплаты и настройки;
- event manager может видеть только статус оплаты внутри назначенной сущности без общих отчётов;
- trainer видит свои выплаты и оплаты назначенных занятий по политике организации;
- viewer не получает финансовые данные по умолчанию.

## Правила целостности

- платёж и участие — разные сущности;
- сумма заказа фиксируется на момент создания;
- изменение цены события не переписывает существующие orders;
- ручная корректировка баланса запрещена;
- финансовые операции сохраняются после архивирования события;
- каждая чувствительная операция создаёт audit event;
- данные платёжной карты не хранятся в VolleyPlay.

## Состояния

- выплаты не подключены;
- проверка реквизитов;
- провайдер недоступен;
- частичные данные;
- транзакция оспаривается;
- валюта не поддерживается;
- permission denied;
- offline read-only.
