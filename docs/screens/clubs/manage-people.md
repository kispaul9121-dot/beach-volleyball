# Кабинет организации — Люди

- Parent Screen ID: `club.manage`
- Route state: `/clubs/:clubId/manage?tab=people`
- Source of truth: `docs/ORGANIZATION_ADMIN.yaml`

## Назначение

Общий рабочий каталог людей, связанных с организацией. Он не заменяет публичные профили и не смешивает спортивные данные с кадровыми или финансовыми правами.

## Категории

- Участники и клиенты;
- Тренеры;
- Сотрудники;
- Заявки и приглашения;
- Заблокированные — только уполномоченным ролям.

Раздел `Сотрудники` показывает краткий список. Полное управление ролями и доступом открывается через `Сотрудники и права`.

## Карточка человека

Общие поля:

- имя и аватар;
- тип связи с организацией;
- активные участия или назначения;
- ближайшее событие;
- задолженность или абонемент — только при наличии финансового права;
- статус приглашения;
- заметка о требуемом действии;
- одно главное действие.

Карточка не показывает медицинские, юридические, приватные тренерские или платёжные данные без отдельного разрешения.

## Карточка клиента / участника

Может включать:

- историю регистраций;
- текущие абонементы;
- посещаемость;
- ожидающие заявки;
- открытые платежи;
- согласия и ограничения, которые пользователь разрешил показывать организации;
- связанные чаты.

Действия: открыть участие, пригласить на событие, написать, открыть оплату, заблокировать взаимодействие по правилам организации.

## Карточка тренера

Показывает:

- публичный профиль;
- статус проверки;
- связанные площадки;
- назначенные группы и занятия;
- доступность;
- роль сотрудника, если существует membership;
- конфликты расписания.

Публичный профиль тренера и кадровая запись остаются разными объектами.

## Поиск и фильтры

- имя, email или телефон — только по разрешённым полям;
- тип связи;
- активность;
- площадка;
- тренер / группа;
- наличие абонемента;
- наличие задолженности;
- дата последнего участия;
- статус приглашения.

## Приватность

- поиск по email и телефону доступен только ролям с соответствующим разрешением;
- личная спортивная статистика не становится собственностью организации;
- приватные тренерские заметки имеют отдельный scope;
- экспорт списка фиксируется в аудите;
- чувствительные поля маскируются в списках;
- потеря membership немедленно закрывает внутренние карточки.

## Действия

- `organization_admin.open_client`;
- пригласить на активность;
- пригласить в группу;
- открыть связанные участия;
- открыть задолженности;
- написать сообщение;
- перейти к управлению сотрудником;
- экспортировать разрешённый список.

## Состояния

- нет людей выбранной категории;
- профиль удалён, история участия сохранена;
- приглашение истекло;
- контактные данные скрыты;
- доступ сотрудника отозван;
- совпадающие профили требуют проверки;
- offline read-only.
