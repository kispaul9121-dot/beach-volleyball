# Кабинет организации — Настройки

- Parent Screen ID: `club.manage`
- Route state: `/clubs/:clubId/manage?tab=settings`
- Source of truth: `docs/ORGANIZATION_ADMIN.yaml`

## Назначение

Настройки конкретной организации. Они не заменяют общие настройки аккаунта и не должны изменяться при переключении на другой личный actor.

## Разделы

### Основное

- название;
- логотип;
- тип организации;
- описание;
- контакты;
- город и часовой пояс;
- языки;
- статус проверки.

### Публичная страница

- опубликованные блоки;
- фотографии;
- площадки;
- тренеры;
- активности;
- туры;
- контакты;
- отзывы;
- предпросмотр публичной страницы.

### Операционные значения по умолчанию

- базовая длительность событий;
- буфер между слотами;
- правила заявок;
- лист ожидания;
- политика отмены;
- шаблон возврата;
- значения уведомлений;
- стандартный ответственный;
- видимость новых сущностей.

Эти настройки подставляются в новый черновик, но не изменяют уже созданные сущности.

### Коммуникации

- отправитель уведомлений;
- шаблоны сообщений;
- каналы;
- тихие часы;
- критические уведомления;
- правила массовых объявлений.

### Платежи и возвраты

- подключение выплат;
- валюта по умолчанию;
- политика возврата;
- внешние способы оплаты;
- финансовые роли;
- реквизиты и документы.

Сами транзакции находятся во вкладке `Финансы`.

### Документы и проверка

- юридические данные;
- документы организации;
- срок действия;
- договоры;
- статус проверки;
- требования к исправлению.

### Интеграции

- внешний календарь;
- экспорт расписания;
- будущие платёжные и бухгалтерские интеграции;
- webhook/API — после MVP.

### Опасная зона

- приостановить публичную страницу;
- перевести организацию в read-only;
- передать владение;
- деактивировать организацию.

## Права

- просмотр и изменение разделяются;
- юридические и платёжные данные доступны только owner, admin с разрешением или finance;
- передача владения доступна только owner;
- сотрудник не может самостоятельно расширить свои права;
- изменения значимых политик создают audit event.

## Сохранение

- простые поля сохраняются по разделу;
- чувствительные изменения требуют повторной аутентификации;
- перед сохранением показываются затронутые сущности;
- конфликт версии открывает сравнение, а не молча перезаписывает изменения другого сотрудника;
- черновик публичной страницы не виден пользователям до публикации.

## Состояния

- профиль организации не завершён;
- проверка ожидается;
- документ отклонён;
- выплаты не подключены;
- несохранённые изменения;
- конфликт версий;
- permission denied;
- организация деактивирована;
- offline read-only.
