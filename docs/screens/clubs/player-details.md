# Публичный профиль игрока

- Screen ID: `player.public_profile`
- Route: `/players/:playerId`
- Parent discovery: `Профиль → Мои игроки`, участники события, поиск игроков
- Data source: public player profile, privacy settings, shared-event summary

## Назначение

Показать разрешённую публичную спортивную информацию об игроке и дать контекстные действия без превращения профиля в социальную сеть.

## Нижняя навигация

Экран использует неизменяемое глобальное меню `Главная · События · Чаты · Клубы · Профиль` и не создаёт второго набора вкладок. Активная вкладка сохраняется из исходного tab stack; прямой deep link возвращается в `Профиль → Мои игроки`.

## Блоки

- имя, фото и город;
- уровень и предпочитаемые форматы;
- игровая рука и позиция — когда заполнены и разрешены;
- публичная статистика;
- общие события с текущим пользователем;
- ближайшие публичные участия — только если игрок разрешил их показывать;
- связанные клубы — только публичные;
- настройки приватности определяют видимость каждого блока.

## Действия

- `players.save_from_public_profile` — добавить в `Мои игроки`;
- `players.remove_from_public_profile` — удалить из `Мои игроки`;
- `players.invite_from_public_profile` — открыть выбор конкретного события для приглашения;
- `player.open_shared_event` — открыть общее событие;
- `player.share_profile` — поделиться публичной ссылкой;
- `player.report_profile` — пожаловаться.

Личного сообщения по умолчанию нет. Оно появляется только при отдельном разрешении пользователя и не создаётся автоматически после добавления в `Мои игроки`.

## Варианты

### Незнакомый игрок

Видит только публичные поля и действие `Добавить в Мои игроки`.

### Сохранённый игрок

Видит отметку `В Моих игроках`, действие удаления и приглашение в конкретное событие.

### Владелец профиля

Видит плашку `Ваш публичный профиль` и действие перехода в настройки приватности; нельзя добавлять самого себя в `Мои игроки`.

## Навигация

- обычный вход: из `Профиль → Мои игроки`, участников события или поиска;
- нажатие на общее событие ведёт на его канонический details screen;
- deep link использует `/players/:playerId`;
- при пустой истории возврат выполняется в `/profile/players`.

## Интеграции

- player directory;
- приватность профиля;
- события и участники;
- жалобы и модерация;
- уведомление об приглашении создаётся только после выбора конкретного события и отправки приглашения.

## Состояния

- loading;
- профиль с минимальным набором публичных данных;
- нет общих событий;
- профиль скрыт владельцем;
- permission denied для закрытых полей;
- профиль удалён;
- error;
- offline с последней доступной публичной версией.
