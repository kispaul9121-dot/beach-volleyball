# Главная

- Screen ID: `home.main`
- Route: `/`
- Variants: `player`, `trainer`, `organization`
- UI contract: `docs/DESIGN_SYSTEM.md` and `docs/DESIGN_TOKENS.yaml`

Главная — персональная оперативная сводка активного профиля. Она не заменяет публичный каталог и показывает только ближайшие действия, события и задачи пользователя.

## Общая верхняя панель

- аватар и название активного профиля → `actor.switcher`;
- уведомления;
- текущий город;
- глобальное действие создания в верхней панели, если активный профиль может создавать сущности;
- корневой экран не показывает кнопку назад.

## Игрок

Порядок блоков:

1. ближайшее участие;
2. действия: оплатить, подтвердить, внести результат;
3. сегодня в календаре;
4. приглашения;
5. мои созданные события — только если игрок что-то организует;
6. рекомендации.

## Тренер

Порядок блоков:

1. тренировки сегодня;
2. посещаемость, которую нужно заполнить;
3. заявки и неоплаченные места;
4. ближайшие занятия;
5. блок `Управляю` с играми, турнирами, сезонами и турами;
6. личные участия тренера как игрока.

Отдельных тренировочных групп в MVP нет. Повторные занятия создаются через шаблон или дублирование тренировки и остаются отдельными событиями.

## Организация

Порядок блоков:

1. операционная сводка сегодня;
2. события, требующие действий;
3. загрузка площадок и кортов;
4. заявки и оплаты;
5. назначенные сотрудники;
6. блок `Управляю`;
7. сообщения и инциденты.

## Карточка управляемой сущности

Показывает:

- плашку `Вы организатор` или роль сотрудника;
- профиль-создатель;
- ближайшее действие;
- участников X из Y;
- задолженности или незакрытые результаты;
- кнопку `Управлять`.

На карточке допускается только одно главное действие. Вся карточка открывает канонический экран сущности или кабинет управления.

## Нижняя навигация

Всегда используется один глобальный компонент и один порядок:

```text
Главная · События · Чаты · Клубы · Профиль
```

На этом экране активна вкладка `Главная`. Профиль игрока, тренера или организации не меняет набор, порядок, названия и маршруты нижнего меню.

## Состояния

- loading;
- empty: нет ближайших действий;
- partial error для отдельного блока;
- offline с последними сохранёнными данными;
- actor without completed profile — карточка завершения профиля.

Любая карточка ведёт на канонический экран или кабинет управления, а не на копию сущности.
