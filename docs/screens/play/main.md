# События

- Screen ID: `play.main`
- Route: `/play`
- Visible bottom-tab label: `События`
- Internal route and screen ID remain `play` for backward compatibility.
- Variants: `player`, `trainer`, `organization`

## Назначение

Единый раздел для поиска, участия и управления играми, тренировками, турнирами, сезонами и турами. Структура одинакова для всех профилей; отличаются выбранная вкладка по умолчанию, порядок блоков и доступные действия.

## Верхняя панель

- активный профиль → actor switcher;
- поиск;
- фильтры;
- список / карта;
- кнопка `Создать` с типами, доступными активному профилю.

## Главный переключатель

На экране всегда присутствуют три вкладки:

```text
Все · Участвую · Управляю
```

### Все

Публичный каталог доступных событий. Здесь показываются только сущности с публичной видимостью целиком. Непубличные игры, тренировки, турниры, сезоны и туры во вкладку `Все` не попадают, даже когда у них есть свободные места. Карточка также показывает статус текущего пользователя или активного actor-профиля.

### Участвую

События, где человек является участником, приглашённым, заявителем или находится в листе ожидания. Здесь отображаются и непубличные события, когда у человека есть соответствующее отношение или приглашение. Спортивное участие связывается с player profile, даже когда активен профиль тренера или организации.

### Управляю

События, созданные активным actor-профилем либо доступные пользователю по роли сотрудника. Вверху показываются задачи, требующие действий, затем опубликованные события и черновики.

## Вкладка по умолчанию

- игрок → `Участвую`, если есть ближайшие участия; иначе `Все`;
- тренер → `Управляю`, если есть занятия или задачи на сегодня; иначе `Участвую`;
- организация → `Управляю`;
- выбранная пользователем вкладка сохраняется отдельно для каждого actor-профиля.

## Фильтры типа события

```text
Все типы · Игры · Тренировки · Турниры · Сезоны · Туры
```

Дополнительные фильтры: дата, расстояние, уровень, формат, цена, свободные места, организатор, статус, площадка и ответственный сотрудник — согласно правам.

## Порядок блоков

### Игрок

1. ближайшие участия;
2. требуют действий: оплата, подтверждение, результат;
3. рекомендации — только во вкладке `Все`;
4. собственные события — во вкладке `Управляю`.

### Тренер

1. занятия сегодня;
2. посещаемость и заявки;
3. тренировки и другие тренерские события;
4. личные участия как игрока;
5. публичный каталог во вкладке `Все`.

### Организация

1. задачи, требующие внимания;
2. события сегодня;
3. предстоящие управляемые активности;
4. черновики;
5. публичный каталог во вкладке `Все`.

## Карточка события

Показывает:

- тип и формат;
- название;
- дату, время и место;
- организатора и тип профиля;
- цену;
- свободные места или заполненность;
- статус пользователя;
- ближайшее необходимое действие.

Плашки:

- `Вы организатор`;
- `Вы участвуете`;
- `Требуется оплата`;
- `Заявка отправлена`;
- `Лист ожидания`;
- `Требует действий`;
- `Черновик`.

## Главное действие карточки

- посторонний → `Присоединиться` / `Подать заявку`;
- участник → `Открыть`, `Оплатить` или другое актуальное действие;
- организатор / сотрудник → `Управлять`;
- черновик → `Продолжить`.

На карточке показывается одно основное действие. Дополнительные доступны через контекстное меню.

## Навигационные правила

- глобальное нижнее меню всегда: `Главная · События · Чаты · Клубы · Профиль`;
- отдельная административная нижняя навигация запрещена;
- карточки открывают канонический detail или manage route;
- возврат восстанавливает вкладку, фильтры и позицию списка;
- переход из организации открывает `Управляю` с actorId организации;
- маршрут `/play` сохраняется, хотя пользовательское название вкладки — `События`.

## Состояния

- loading skeleton;
- пустое `Участвую` с переходом в `Все`;
- пустое `Управляю` с кнопкой создания;
- нет результатов фильтра;
- геолокация недоступна;
- offline read-only;
- частичная ошибка карты;
- actor permission changed.
