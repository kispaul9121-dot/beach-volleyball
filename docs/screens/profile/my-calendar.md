# Мой календарь

- Screen ID: `profile.calendar`
- Route: `/profile/calendar`
- Parent: `home.main`

## Назначение

Единый спортивный и операционный календарь аккаунта. Пользователь может видеть события всех профилей либо только активного actor.

## Переключатели

- День / Неделя / Месяц;
- Все профили / Активный профиль;
- Участвую / Управляю / Всё;
- фильтр типов сущностей.

## Типы элементов

- одноразовая игра;
- тренировка;
- турнир;
- игровой день сезонного турнира;
- тур;
- бронирование корта;
- дедлайн оплаты;
- дедлайн регистрации;
- задача организатора;
- смена сотрудника организации.

## Карточка

- actor-профиль цветной меткой или аватаром;
- тип и статус;
- роль пользователя: участник, тренер, организатор, сотрудник;
- время и место;
- главное действие.

Примеры действий: Открыть, Управлять, Оплатить, Отметить посещаемость, Внести результат, Подтвердить перенос.

## Представление тренера

Дополнительно показывает занятия, окна расписания и незаполненную посещаемость.

## Представление организации

Дополнительно показывает корты, сотрудников, пересечения площадок и операционные задачи.

## Взаимодействия

- тап → канонический экран или кабинет управления согласно роли;
- свайп по датам;
- кнопка `Сегодня`;
- открытие фильтров;
- экспорт во внешний календарь;
- открыть чат;
- отмена участия через подтверждение.

## Конфликты

Календарь предупреждает, если один user_account одновременно участвует или назначен в пересекающихся событиях разных actor-профилей. Конфликт не всегда блокирует организационные задачи, но должен быть видим.

## Состояния

- пустой день / период;
- только скрытые фильтрами события;
- внешний календарь не подключён;
- событие изменено;
- actor потерял доступ;
- offline с кэшированными данными.

Назад → Профиль. При deep link fallback `/`.
