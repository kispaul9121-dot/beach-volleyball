# Платежи

- Screen ID: `profile.payments`
- Route: `/profile/payments`
- Variants: `player`, `trainer`, `organization`

## Игрок

Вкладки:

- К оплате;
- Оплачено;
- Возвраты;
- Все.

Карточка: сумма, сущность, организатор, срок оплаты, статус, чек. Действия: Оплатить, Детали, Запросить возврат, Открыть событие.

## Тренер

Вкладки:

- Мои оплаты как участника;
- Поступления;
- Ожидаемые выплаты;
- Возвраты.

Дополнительно показываются подключение выплат, удержания платформы и события-источники.

## Организация

Вкладки:

- Поступления;
- Задолженности;
- Возвраты;
- Выплаты;
- Отчёты.

Фильтры: период, событие, площадка, сотрудник, статус.

## Правила

- участие и платёж являются разными сущностями;
- оплаченный статус приходит от платёжного провайдера;
- изменение цены не изменяет уже созданный заказ без явной операции;
- возврат создаёт новую транзакцию;
- финансовая история не удаляется вместе с событием;
- пользователь видит только свои платежи, сотрудник — согласно финансовой роли.

## Состояния

- выплаты не подключены;
- проверка реквизитов;
- платёж обрабатывается;
- платёж не прошёл;
- частичный возврат;
- спор;
- offline read-only.
