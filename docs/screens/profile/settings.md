# Настройки

- Screen ID: `profile.settings`
- Route: `/profile/settings`
- Variants: `player`, `trainer`, `organization`
- Source of truth: `docs/PROFILE_SETTINGS.yaml`

## Назначение

Один экран настроек для одного аккаунта с динамической профильной частью. Общие настройки аккаунта не меняются после переключения профиля. Профильные разделы перестраиваются под игрока, тренера или организацию.

## Навигация

Глобальная нижняя навигация сохраняется:

```text
Главная · Играть · Чаты · Клубы · Профиль
```

Активна вкладка `Профиль`.

## Шапка

- стрелка назад;
- заголовок `Настройки`;
- помощь;
- карточка активного профиля;
- аватар или логотип;
- имя или название;
- тип профиля;
- статус `Активный профиль`;
- кнопка `Сменить профиль`.

При наличии нескольких профилей внутри карточки показывается компактный selector:

```text
Игрок · Тренер · Организация
```

Переключение обновляет только профильные секции и связанные настройки. Аккаунт остаётся тем же.

## Общая секция «Аккаунт»

Одинакова для всех вариантов:

- Email и вход;
- Пароль и безопасность;
- Активные устройства;
- Язык и тема.

Каждая строка открывает отдельный экран или системный flow. На главном экране нет формы редактирования.

## Вариант «Игрок»

Секция `Активный профиль — Игрок`:

- Личные данные;
- Уровень и рейтинг;
- Игровая сторона;
- Предпочитаемые форматы;
- Кто может приглашать — с текущим значением;
- Кто может писать — с текущим значением;
- Видимость статистики — с текущим значением.

Секция `Уведомления`:

- Общие уведомления;
- Уведомления игрока;
- Чаты и сообщения.

Секция `Приватность`:

- Видимость профиля;
- Заблокированные пользователи.

## Вариант «Тренер»

Секция `Активный профиль — Тренер`:

- Публичный профиль;
- Специализации и опыт;
- Расписание и занятость;
- Правила записи;
- Правила отмены;
- Выплаты тренера.

Секция `Уведомления`:

- Общие уведомления;
- Уведомления тренера;
- Чаты и сообщения.

Секция `Приватность`:

- Кто может писать;
- Кто может приглашать;
- Видимость профиля;
- Видимость статистики.

## Вариант «Организация»

Секция `Активный профиль — Организация`:

- Публичные данные клуба;
- Уведомления организации;
- Моя роль и права — с текущей membership role;
- Открыть настройки организации.

`Открыть настройки организации` ведёт в:

```text
/clubs/:clubId/manage?section=settings
```

Полная конфигурация организации — сотрудники, роли, площадки, юридические данные, финансы, политики, передача владения и деактивация — не дублируется в Профиле.

Секция `Уведомления`:

- Общие уведомления;
- Уведомления организации;
- Чаты и сообщения.

Секция `Приватность`:

- Кто может писать;
- Кто может приглашать;
- Видимость клуба;
- Заблокированные пользователи.

## Общая секция «Приложение»

Одинакова для всех вариантов:

- Разрешения приложения;
- Поддержка;
- О приложении.

## Опасные действия

Внизу отдельный визуально опасный блок:

- Выйти из аккаунта;
- Удалить аккаунт.

Обе операции требуют подтверждения. Удаление аккаунта дополнительно проверяет активные события, задолженности, выплаты и владение организациями.

Удаление отдельного профиля не показывается в этом блоке. Оно выполняется через `Профиль → Мои профили` с собственными проверками зависимостей.

## Уведомления и наследование

Приоритет настроек:

```text
конкретное событие
→ активный профиль
→ аккаунт
→ системное значение по умолчанию
```

Самая конкретная настройка имеет приоритет.

## Переключение профиля

- при чистом состоянии профиль меняется сразу, экран остаётся открыт;
- профильные секции обновляются на месте;
- общие секции не исчезают и не меняют назначение;
- при открытой дочерней форме с несохранёнными данными предлагается сохранить, отбросить или отменить переключение;
- при отсутствии права организации показывается роль и понятное ограничение;
- повторный вход не требуется.

## Визуальные правила

- один экран — один уровень меню;
- внутри строк не размещать формы и несколько кнопок;
- короткое текущее значение допустимо справа;
- карточка активного профиля визуально отделена от списка;
- у каждого actor-типа свой семантический акцент, но общая структура одинакова;
- не показывать более семи строк в одной карточке;
- опасные действия отделены цветом и находятся внизу.

## Состояния

- loading;
- только профиль игрока;
- несколько профилей;
- профиль заполнен не полностью;
- проверка тренера или организации;
- permission denied;
- offline read-only;
- ошибка сохранения на дочернем экране.
