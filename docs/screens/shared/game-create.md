# Создание игры

- Screen ID: `game.create`
- Route: `/games/create`
- MVP contract: `docs/GAME_MVP.yaml`
- Join contract: `docs/JOIN_FLOW.yaml`

Мастер открывается только из раздела `Игры → Режим управления → + Создать → Создать игру`.

## Постоянная шапка

- закрыть;
- прогресс `1 из 4`;
- active actor, от имени которого создаётся игра;
- состояние автосохранения.

Выбор actor не является отдельным шагом: он показывается в шапке и при необходимости меняется через actor switcher.

## Шаг 1. Что и когда

- название;
- необязательное описание;
- дата и время начала;
- время окончания;
- часовой пояс;
- публичная или приватная игра.

## Шаг 2. Место и формат

- сохранённая площадка, клуб/корт или ручной адрес;
- количество зарезервированных кортов;
- формат: `2×2`, `4×4`, `Смешанные пары` или `Матч фиксированных команд`;
- поле ввода `Количество игроков`.

Количество игроков вводит организатор вручную. Оно не должно быть только автоматически рассчитанным значением.

При вводе выполняются три проверки:

1. значение не ниже минимального для выбранного формата;
2. значение не превышает лимит выбранной площадки или забронированного слота;
3. значение соблюдает дополнительные правила формата, когда они есть.

Для площадки используется наиболее точный доступный лимит:

- лимит конкретного забронированного слота;
- настроенный лимит площадки;
- при ручном адресе — лимит, указанный организатором.

При превышении публикация блокируется:

```text
Выбранная площадка рассчитана максимум на 20 игроков
```

Правила форматов:

- `2×2` — минимум 4 игрока на корт; большее число допускается для ротации или резерва;
- `4×4` — минимум 8 игроков на корт; большее число допускается для замен или резерва;
- `Смешанные пары` — минимум 4 игрока, верхний предел задаётся только вместимостью площадки; число не обязано быть кратно пяти;
- `Матч фиксированных команд` — минимум две команды, итоговое число сверяется с размером команды.

Для `Смешанных пар` система создаёт столько матчей, сколько указано игроков:

```text
5 игроков  → 5 матчей
8 игроков  → 8 матчей
15 игроков → 15 матчей
```

В каждом матче участвуют две пары. Генератор стремится дать каждому игроку по четыре матча и равномерно распределить партнёров, соперников и время ожидания. Количество кортов влияет на расписание и число одновременных матчей, но не изменяет общее количество строк матчей.

Технический идентификатор этого формата пока остаётся `rotation_five` для совместимости, но продуктово он больше не означает ровно пять игроков.

## Шаг 3. Участие и цена

Способ подтверждения и способ оплаты настраиваются независимо.

```text
Как игроки вступают?
○ Сразу записываются
○ Отправляют заявку
○ Только по приглашению
```

```text
Стоимость
○ Бесплатно
○ Оплата онлайн
○ Оплата организатору вне приложения
```

Дополнительно:

- цена, когда она требуется;
- лист ожидания при заполнении;
- необязательные ограничения по уровню, полу и возрасту.

Бесплатная игра может требовать одобрения организатора. Платная игра может подтверждаться сразу либо после заявки. Полная матрица результата находится в `docs/JOIN_FLOW.yaml`.

## Шаг 4. Проверка и публикация

Показываются:

- компактный preview игры;
- место, корты и лимит игроков;
- формат;
- количество будущих матчей для автоматически генерируемого режима;
- способ вступления;
- стоимость;
- ошибки, блокирующие публикацию.

Действия:

```text
[Опубликовать]
[Сохранить черновик]
```

После публикации открывается `/games/:gameId/manage`. После сохранения и закрытия — `/play?mode=manage&category=games`.

## Дополнительные настройки

Очки за матч или сет, рейтинг, порядок генерации и распределение по кортам не создают отдельные шаги. Они находятся в свёрнутом блоке `Дополнительные настройки` внутри соответствующего шага.

## Профиль-создатель

### Игрок

Показывается как частный организатор. Может создать бесплатную или платную социальную игру. Онлайн-оплата требует подключения выплат.

### Тренер

Можно отметить `игра с тренером` и показывать профессиональный профиль.

### Организация

Можно использовать собственную площадку, назначенного сотрудника, бренд и финансовые параметры организации.

## Черновик

- сохраняется автоматически;
- хранит actorId;
- показывается в `Игры → Режим управления → Игры → Черновики`;
- смена actor требует подтверждения и повторной проверки возможностей;
- количество игроков и лимит площадки сохраняются как отдельные значения.

## Состояния

- loading venue limits;
- autosaving;
- saved;
- validation error;
- venue capacity exceeded;
- format capacity mismatch;
- payout configuration required;
- offline draft only;
- publish in progress;
- publish failed.
