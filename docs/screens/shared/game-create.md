# Создание игры

- Screen ID: `game.create`
- Route: `/games/create`
- MVP contract: `docs/GAME_MVP.yaml`
- Join contract: `docs/JOIN_FLOW.yaml`

Мастер открывается только из `Игры → Режим управления → + Создать → Создать игру`.

## Постоянная шапка

- закрыть;
- прогресс `1 из 4`;
- active actor;
- состояние автосохранения.

## Шаг 1. Что и когда

- название;
- необязательное описание;
- дата и время начала;
- время окончания;
- часовой пояс;
- публичная или приватная игра.

## Шаг 2. Место и формат

- площадка или ручной адрес;
- количество зарезервированных кортов;
- формат: `2×2`, `4×4`, `Смешанные пары на 5 игроков` или `Матч фиксированных команд`;
- `Количество игроков`;
- `Количество матчей`, когда формат этого требует.

Количество игроков вводит организатор вручную и оно проверяется по формату и лимиту площадки.

### Смешанные пары на 5 игроков

В этой разовой игре количество участников всегда равно пяти:

```text
Количество игроков: 5
```

Публикация блокируется при любом другом значении:

```text
Для этого формата требуется ровно 5 игроков
```

Количество матчей вводится отдельно:

```text
Количество матчей: 15
```

Поле принимает положительное целое число. Значение по умолчанию — `15`.

Для пяти игроков существует 15 уникальных составов матча: выбирается один отдыхающий игрок, а остальные четверо делятся на две пары тремя способами.

```text
1–14 матчей  → сбалансированная часть полного цикла
15 матчей    → все уникальные составы по одному разу
больше 15   → следующий перемешанный цикл с неизбежными повторами
```

При значении больше 15 показывается предупреждение, но публикация не блокируется.

Десять и пятнадцать участников не создаются как одна разовая игра. Это отдельный турнир **`Тунисский формат`**:

```text
10 игроков → 2 площадки по 5
15 игроков → 3 площадки по 5
```

На каждой площадке проводится собственный цикл смешанных пар. После цикла лидер нижней площадки поднимается выше, а последний игрок верхней площадки опускается ниже. Контракт: `docs/TOURNAMENT_COURT_LADDER.yaml`.

### Другие форматы

- `2×2` — минимум 4 игрока на корт;
- `4×4` — минимум 8 игроков на корт;
- фиксированные команды — минимум две команды.

При превышении лимита площадки публикация блокируется:

```text
Выбранная площадка рассчитана максимум на 20 игроков
```

## Шаг 3. Участие и цена

```text
Как игроки вступают?
○ Сразу записываются
○ Отправляют заявку
○ Только по приглашению
```

```text
Стоимость
○ Бесплатно
○ Оплата онлайн
○ Оплата организатору вне приложения
```

Дополнительно: лист ожидания и необязательные ограничения по уровню, полу и возрасту.

## Шаг 4. Проверка и публикация

Показываются:

- preview игры;
- место и корты;
- количество игроков;
- количество матчей;
- формат;
- способ вступления;
- стоимость;
- блокирующие ошибки и предупреждения.

```text
[Опубликовать]
[Сохранить черновик]
```

После публикации открывается `/games/:gameId/manage`. После сохранения и закрытия — `/play?mode=manage&category=games`.

## Дополнительные настройки

Очки за матч или сет, рейтинг, порядок генерации и распределение по кортам находятся в свёрнутом блоке и не создают отдельные шаги.

## Черновик

- сохраняется автоматически;
- хранит actorId;
- сохраняет количество игроков и количество матчей как разные поля;
- показывается в `Игры → Режим управления → Игры → Черновики`.

## Состояния

- loading venue limits;
- autosaving;
- validation error;
- venue capacity exceeded;
- exact five-player validation failed;
- match count warning after unique cycle;
- payout configuration required;
- offline draft only;
- publish in progress;
- publish failed.
