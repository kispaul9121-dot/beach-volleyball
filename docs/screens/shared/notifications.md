# Уведомления

- Screen ID: `global.notifications`
- Route: `/notifications`

## Категории

- Требуют действий;
- События;
- Сообщения;
- Платежи;
- Системные.

## Карточка

- тип и иконка;
- actor-профиль, к которому относится уведомление;
- связанная сущность;
- время;
- прочитано / непрочитано;
- главное действие.

## Примеры действий

- оплатить;
- подтвердить перенос;
- принять приглашение;
- проверить заявку;
- опубликовать раунд;
- внести результат;
- открыть чат;
- завершить профиль.

## Правила

- уведомление хранит целевой `action_id`, а не только текст;
- при переключении actor можно фильтровать уведомления активного профиля;
- критичные уведомления не исчезают после прочтения, пока действие не выполнено или не стало неактуальным;
- удалённая сущность открывает понятное состояние, а не ошибку маршрута.

## Состояния

- нет уведомлений;
- только прочитанные;
- действие уже выполнено на другом устройстве;
- потеря прав;
- offline.
