# Добавить игроков

- Screen ID: `player.picker`
- Route: `system://player-picker`
- Navigator: full-screen modal
- Variants: `create_context`, `manage_context`
- Source of truth: `docs/PLAYER_DIRECTORY.yaml`

## Назначение

Единый модальный экран множественного выбора игроков для игры, тренировки, турнира, сезона или тура.

Экран открывается только из мастера создания или управления конкретным событием. Он не является отдельным разделом приложения и не заменяет `Профиль → Мои игроки`.

## Навигация

- глобальная нижняя навигация скрыта;
- закрытие возвращает на тот же шаг создания или управления;
- выбранные игроки сохраняются локально при временном закрытии, пока пользователь не покинул мастер;
- deep link на picker без контекста события запрещён;
- `returnTo` обязан указывать на исходный мастер или экран управления.

## Входной контекст

Обязательные параметры:

- `entity_type`: `game | training | tournament | season | tour`;
- `entity_id` или `draft_id`;
- `actor_id` активного профиля, который приглашает;
- `return_to`;
- `invitation_permission`;
- текущая вместимость и количество доступных мест.

Опциональные параметры:

- уже выбранные игроки;
- уже приглашённые и подтверждённые участники;
- режим регистрации события;
- цена и признак обязательной оплаты;
- лист ожидания;
- разрешение `add_as_confirmed`.

## Верхняя часть

- действие закрытия `players.close_picker`;
- заголовок `Добавить игроков`;
- название и дата события;
- строка `Свободно N мест`, когда вместимость ограничена;
- поиск по текущему источнику — `players.picker_search`.

## Источники

Переключатель:

```text
Мои игроки · Недавние · Поиск
```

- `Мои игроки` — сохранённые игроки активного actor-профиля;
- `Недавние` — игроки из доступных общих событий;
- `Поиск` — публичные player-профили с учётом приватности, блокировок и антиспама.

Переключение источника использует `players.picker_change_source` и не сбрасывает выбор.

## Строка игрока

Показывает:

- аватар;
- имя;
- публичный уровень и город, когда доступны;
- последний общий контекст, когда разрешён;
- состояние выбора;
- статус относительно текущего события.

Возможные статусы:

- доступен;
- выбран;
- уже приглашён;
- подтверждён;
- отказался;
- в листе ожидания;
- недоступен из-за заполненной вместимости;
- заблокирован или скрыт политикой приватности.

Нажатие использует `players.picker_toggle_player`.

## Нижняя панель

Показывается после выбора хотя бы одного игрока:

- `Выбрано: N`;
- прогноз заполнения `X из Y`;
- основное действие `Отправить приглашения` — `players.send_invitations`;
- вторичное действие `Добавить как подтверждённых` — `players.add_as_confirmed`, только при наличии permission.

`Добавить как подтверждённых` требует подтверждения офлайн-договорённости, причины и audit event для каждого игрока.

## Правила вместимости

- нельзя выбрать больше игроков, чем позволяет оставшаяся вместимость, если лист ожидания выключен;
- при включённом листе ожидания интерфейс заранее показывает, кто попадёт в ожидание;
- уже подтверждённые участники не уменьшаются и не заменяются молча;
- при изменении вместимости во время открытого picker сервер повторно валидирует отправку;
- конфликт показывает понятный результат по каждому игроку, а не общий неопределённый error.

## Приглашения и оплата

- выбор игрока ещё не создаёт участие;
- отправка создаёт отдельное приглашение на конкретное событие;
- приглашённый не получает доступ к чату до подтверждения участия;
- для платного события принятие приглашения переводит участие в `payment_required`, если оплата обязательна;
- отправка приглашения не отмечает платёж как выполненный;
- повторная отправка существующего активного приглашения запрещена или превращается в явное напоминание согласно политике события.

## Массовая отправка

Перед отправкой показывается сводка:

- количество новых приглашений;
- уже приглашённые;
- недоступные;
- ожидающие листа ожидания;
- платные места.

Массовая отправка проверяет rate limit, антиспам, actor ownership и permission на событие.

## Действия

- `players.close_picker`;
- `players.picker_change_source`;
- `players.picker_search`;
- `players.picker_toggle_player`;
- `players.send_invitations`;
- `players.add_as_confirmed`.

## Состояния

- loading;
- пустой список `Мои игроки`;
- нет недавних игроков;
- поиск ничего не нашёл;
- offline read-only;
- permission denied;
- событие заполнено;
- часть игроков стала недоступна;
- приглашение уже существует;
- rate limit;
- отправка частично успешна;
- отправка успешна.

## Успешное завершение

После отправки:

1. показать результат по каждому выбранному игроку;
2. обновить счётчики приглашённых и свободных мест;
3. закрыть modal;
4. вернуть пользователя на исходный шаг или экран управления;
5. сохранить `returnTo` и actor-контекст без переключения профиля.
