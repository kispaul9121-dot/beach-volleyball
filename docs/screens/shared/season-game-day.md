# Игровой день сезона

## season.game_day

- Route: `/seasons/:seasonId/game-days/:gameDayId`

Участник видит:

- дату, время и место;
- регистрацию на день;
- корт и раунд;
- партнёра и соперников;
- отдых в ротации пяти;
- результаты;
- вклад дня в таблицу сезона;
- объявления и чат.

Вкладки: Обзор, Раунды / Матчи, Участники, Результаты.

## season.game_day_manage

- Route: `/seasons/:seasonId/game-days/:gameDayId/manage`

### Настройки дня

- площадка и корты;
- `court_count`;
- вместимость;
- время;
- регистрация;
- цена дня;
- игровой формат;
- движение между кортами.

### Участники

Регистрация, подтверждение, резерв, лист ожидания, check-in, замены.

### Распределение

- сформировать группы;
- проверить уровень / рейтинг;
- ручная корректировка;
- конфликты;
- сохранить черновик.

### Раунды

- генерация по `GAME_FORMATS.yaml`;
- предпросмотр всех кортов;
- публикация;
- запуск и завершение;
- откат с причиной.

### Результаты

Ввод по корту и раунду, пропущенные значения, пересчёт личных очков и сезонной таблицы.

## Инварианты ротации пяти

- участников ровно `court_count × 5` для полного цикла;
- в каждой группе пять игроков;
- каждый отдыхает один раз;
- каждый играет в паре с каждым один раз;
- один игрок не может быть назначен на два корта в одном раунде.

Если участников меньше, организатор должен выбрать: уменьшить число кортов, оставить резервную неполную группу без автоматической генерации или изменить формат. Система не должна молча создавать некорректное расписание.
